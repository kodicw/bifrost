# 👻 Bifrost: Product Requirements Document (PRD)

## 1. Overview
**Bifrost** is a NixOS-inspired, declarative configuration manager for Windows. It aims to bridge the gap between the immutable, declarative world of Nix/NixOS and the imperative, stateful nature of Windows. By using a single JSON file as the source of truth, Bifrost allows users to define the desired state of a Windows machine and reconcile it with a single command.

## 2. Problem Statement
Configuring Windows machines—especially virtualized hosts for container orchestration (like the **Imperative Containment** project)—is traditionally a manual, imperative process involving multiple reboots, GUI menus, and brittle PowerShell scripts. Existing tools like Ansible or Chef are often too heavy for minimal VM bootstrapping, while PowerShell DSC (Desired State Configuration) is complex to author and maintain without a domain controller.

## 3. Goals & Objectives
- **Declarative Simplicity**: Use a minimal JSON schema to define system state.
- **Bootstrapping Efficiency**: Enable a "one-liner" installation and configuration process via GitHub/URL.
- **Container Host Optimization**: Provide first-class support for configuring Windows process isolation container hosts.
- **Nix Integration**: Serve as the "bridge" (Bifrost) that allows Nix to declaratively manage internal Windows VM states.
- **Idempotency**: Ensure that running the script multiple times results in the same system state without side effects.

## 4. Target Audience
- Developers using **Nix/NixOS** to manage virtualized Windows environments.
- Systems Engineers deploying Windows-based container hosts.
- Power users seeking a "dotfiles-style" approach to Windows configuration.

## 5. Functional Requirements (Current Modules)
- **Users**: Create and manage local accounts.
- **Packages**: Manage software via Scoop (Local and Global).
- **System**: Enable/disable Windows Optional Features and Capabilities.
- **Networking**: Manage firewall rules with a "Pure" mode for purging unmanaged rules.
- **Registry**: Enforce registry keys and values (HKLM/HKCU).
- **Services**: Manage service state (Running/Stopped) and startup types.
- **Files**: Declaratively write text files to disk with specific encoding.
- **Downloads**: Fetch remote assets (URLs) to local paths.
- **Scripts**: Execute post-provisioning PowerShell logic.

## 6. Technical Specifications
- **Language**: PowerShell 5.1+ (Native Windows Compatibility).
- **Configuration Format**: JSON (standard, easily generated by Nix).
- **Execution Model**: 
    - **Impure**: Apply state, leave unmanaged entities alone.
    - **Pure**: Purge specific unmanaged states (currently Firewall) before applying.
- **Remote Support**: Capability to fetch JSON configuration from any HTTP(S) endpoint.

## 7. Future Roadmap
- **Nix-Native Mapping**: Implement features to mount/write directories from Nix store paths directly into the Windows filesystem via VirtIO-FS or SMB.
- **Lock Files**: Implement a `bifrost.lock.json` to pin package versions and system states.
- **Native Container Host Profiles**: Pre-defined "flavors" for optimized Windows Server/Windows 11 container hosting.
- **State Auditing**: A "Dry-Run" mode to show what changes would be made without applying them.

## 8. Non-Goals
- Replacing enterprise-grade fleet management (Intune/SCCM).
- Managing Windows Updates (deferred to native OS mechanisms for now).
- Supporting legacy Windows versions (prior to Windows 10/Server 2016).
